数据库

关系型和非关系型数据库区别

关系型数据库最典型的数据结构是表，由二维表及其之间的联系所组成的一个数据组织。

优点：

​ 1、易于维护：都是使用表结构，格式一致；

​ 2、使用方便：SQL语言通用，可用于复杂查询；

​ 3、复杂操作：支持SQL，可用于一个表以及多个表之间非常复杂的查询。

缺点：

​ 1、读写性能较差，尤其是海量数据的高效率读写；

​ 2、固定的表结构，灵活度稍欠；

​ 3、不能满足高并发读写需求，对于传统关系型数据库来说，磁盘IO是一个很大的瓶颈。

非关系型数据库从严格意义上来说不是一种数据库，应该是一种数据结构化存储方式的集合，可以是文档或键值对等。

优点：

​ 1、格式灵活：存储的格式可以是<K,V>的形式、文档形式、图片形式

​ 2、速度快：nosql可以使用磁盘或随机存储器作为载体，而关系型数据库是能使用硬盘；

​ 3、高扩展性；

​ 4、成本低：nosql数据库部署简单，基本都是开源软件。

缺点：

​ 1、不提供sql支持，学习和使用成本较高；

​ 2、无事务处理；

​ 3、数据结构相对复杂，复杂查询方面稍欠。

Innodb和MyISAM区别

区别：

1、Innodb是事务安全的，MyISAM非事务安全

2、Innodb支持行级锁，MySIAM锁粒度为表级

3、Innodb支持外键，MyIASM不支持外键

4、Innodb不支持全文类型索引，MyISAM支持全文索引

5、Innodb没有保存表的行数，MyISAM保存了表的行数

insert和update多，选用Innodb，select多，选用MyISAM

三大范式

第一范式：关系型数据库默认支持第一范式。即表中的列具有原子性，不能再分割；通俗理解就是一个字段只存储一项信息。

第二范式：在满足第一范式的前提下，要求表中每一个记录或行必须能被唯一区分。

第三范式：在满足第二范式的前提下，要求一个数据库表中不包含已在其他表中定义的非主键字段。

Spring AOP详解

1、AOP 作用？

我们学Java面向对象的时候，如果代码重复了怎么办啊？？可以分成下面几个步骤：

  • 1：抽取成方法
  • 2：抽取类

抽取成类的方式我们称之为：纵向抽取

  • 通过继承的方式实现纵向抽取

但是，我们现在的办法不行：即使抽取成类还是会出现重复的代码，因为这些逻辑(开始、结束、提交事务)依附在我们业务类的方法逻辑中！

现在纵向抽取的方式不行了，AOP的理念：就是将分散在各个业务逻辑代码中相同的代码通过横向切割的方式抽取到一个独立的模块中！

2、AOP 原理

​ AOP底层原理就是基于动态代理！！

​ Spring中动态代理分为两种：

​ 1、JDK 动态代理

​ 2、cglib动态代理

​ JDK动态代理是需要实现某个接口，而我们的类未必全部都有接口，于是就有了cglib动态代理。

​ · cglib代理生成的动态代理对象是目标类的子类

​ · Spring AOP 默认使用JDK 动态代理，如果代理的类没有接口则会使用cglib代理

​ 两种方式的应用选择：如果是单例最好使用cglib，多例最好选择JDK代理。因为JDK在创建代理对象时的性能高于cglib代理，而生成代理对象的运行时性能却比cglib低。

看到这里，将AOP原理总结成一句话：将相同逻辑的重复代码横向抽取出来，使用动态代理技术将这些重复代码织入到目标对象方法中，实现和原来一样的功能。

3、AOP的术语

​ Join point(连接点):Spring AOP是基于动态代理的，所以是方法拦截的，每个成员方法都可以称之为连接点

Point cut(切入点)：我们具体定位到某一个方法，这个方法就称之为切入点

Advice（通知/增强）:表示添加到切点的一段逻辑代码，并定位连接点的方位信息。简单来说，就是定义了干什么，具体是在那干。Spring AOP 提供了5种Advice类型：Before Advice（前置通知）、after returning advice、after throwing advice、after finally advice、around advice。

Weaving（织入）：把advice添加到目标类的具体连接点的过程

Introduction（引介）：允许我们向现有的类添加新的方法或属性，是一种特殊的增强

Aspect（切面）：切面由切入点和advice 组成，它既包括了横切逻辑的定义，也包括了连接点的定义

title： ConcurrentHashMap实现原理

1、为什么在高并发下使用ConcurrentHashMap？

​ A: hashmap在高并发环境下执行put操作会造成Entry链表形成环形数据结构，从而导致next节点永远不为空，因而产生死循环获取Entry；

​ hashtable虽然是线程安全的，但是效率低下，当一个线程访问hashtable的同步方法时，其他线程如果也访问该hashtable，那么会进入阻塞或者轮训状态；

​ JDK1.8之前使用分段锁segment将hashmap分为多个segnment，每个segment分配一个锁，当一个线程占用锁访问其中一段数据时，其他数据同时能被其他线程访问；JDK1.8中采用CAS+synchronized锁实现同步。

HashMap实现原理

1、1 HashMap底层数据结构

​ 数组+链表+红黑树结构。

1、2 Hash算法

Hash算法是将key的hashCode值的高16位与低16位进行异或操作。

1、3 Node<K,V>类

​ Node<K,V>是HashMap内部类实现Map.Entry<K,V>接口，HashMap中哈希桶数组存放的键值对就是Node<K,V>对象。类中维护了一个next指针指向下一个对象。当链表中的元素数量超过TREEIFY_THRESHOLD 时，链表结构会自动转换为红黑树结构。此时，该下标的元素将会转换成TreeNode<K,V>，继承于LinkedHashMap.Entry<K,V>，而LinkedHashMap.Entry<K,V>时Node.Entry<K,V>的子类，因此，hashMap底层数组数据类型即为Node<K，V>.

1、4 put方法实现

​ 1、判断数组是否为空，如果为空，就创建一个table，并获取它的长度；

​ 2、判断计算出来的索引位置是否有值，如果没有就直接放入。如果有数据，则判断key是否相同，如果相同，就覆盖原来的value；

​ 3、判断是否为红黑树，是则直接插入数中；

​ 4、如果不是红黑树，则遍历每个节点，将Node加入到链表末尾，判断链表是否大于8，如果大于，就转换为红黑树。

​ 5、如果没有key发生变化，则modCount不变

1.5 get方法实现

​ 1、判断key是否为null，如果为null，直接返回null

​ 2、判断数组索引位置处的key是否相等，如果相等，取出value

​ 3、判断是否是为红黑树，从树中取值

​ 4、遍历链表查找

1、6 扩容机制

​ 我们使用的是2次幂的扩展(指长度扩为原来2倍)，所以，元素的位置要么是在原位置，要么是在原位置再移动2次幂的位置。看下图可以明白这句话的意思，n为table的长度，图（a）表示扩容前的key1和key2两种key确定索引位置的示例，图（b）表示扩容后key1和key2两种key确定索引位置的示例，其中hash1是key1对应的哈希与高位运算结果。
